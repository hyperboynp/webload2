## üí° Best Practices

### 1. Contrast Ratios
Ensure sufficient contrast between:
- Text and background (minimum 4.5:1 for body text)
- Links and background (minimum 3:1)
- Buttons and background (minimum 3:1)

**Tool:** Use [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/)

---

### 2. Color Harmony

**Complementary Colors:**
- Use complementary colors for primary/secondary
- Example: Orange (#F58220) + Blue (#0066cc)

**Analogous Colors:**
- Use adjacent colors for subtle variations
- Example: Orange (#F58220) + Red-Orange (#e74c3c)

**Triadic Colors:**
- Use three colors equally spaced on color wheel
- Example: Orange + Green + Purple

---

### 3. Brand Consistency

**Maintain Consistency:**
- Primary color should be your main brand color
- Use throughout buttons, links, CTAs
- Reserve for important actions

**Accent Sparingly:**
- Secondary color for highlights
- Don't overuse - keep it impactful

---

### 4. Accessibility

**Consider Users With:**
- Color blindness (use contrast, not just color)
- Low vision (ensure high contrast)
- Screen readers (proper semantic HTML)

**Guidelines:**
- WCAG 2.1 Level AA compliance
- 4.5:1 contrast for normal text
- 3:1 contrast for large text (18pt+)

---

### 5. Testing

**Test On:**
- ‚úÖ Desktop (Chrome, Firefox, Safari, Edge)
- ‚úÖ Mobile (iOS Safari, Android Chrome)
- ‚úÖ Tablet (iPad, Android tablets)
- ‚úÖ Different screen brightness
- ‚úÖ With color blindness simulator

## üîß Troubleshooting

### Colors Not Appearing?

**Solution 1:** Clear cache
```
1. Clear browser cache (Ctrl+Shift+Delete)
2. If using cache plugin: Clear WordPress cache
3. Hard refresh page (Ctrl+F5 or Cmd+Shift+R)
```

**Solution 2:** Check specificity
- Custom CSS may be overriding colors
- Check browser DevTools (F12) for conflicting styles

**Solution 3:** Verify theme version
- Ensure you're running updated theme version
- Check if functions.php has latest code

---

### Colors Look Different on Mobile?

**Possible Causes:**
- Screen calibration differences
- Mobile browser rendering
- Night mode/Dark mode active

**Solution:**
- Test on actual devices
- Use standard web-safe colors
- Ensure sufficient contrast

---

### Footer Icons Wrong Color?

**Check:**
1. Footer Icon Color setting in Theme Colors
2. SVG stroke attribute in code
3. CSS specificity issues

**Solution:**
```
Go to: Appearance ‚Üí Customize ‚Üí Theme Colors
Find: Footer Icon Color
Set: Your desired color (e.g., #d4a747)
```

## üéì Advanced Tips

### 1. Use CSS Variables
All colors are output as CSS variables in `:root`:
```css
:root {
  --primary-orange: #F58220;
  --primary-orange-hover: #e07419;
  --bg-green: #3CB878;
  --text-dark: #1a1d29;
  --text-gray: #6c757d;
  --bg-light-gray: #f8f9fa;
}
```

You can use these in custom CSS!

### 2. Create Color Presets
Save your favorite color schemes in a document for quick switching between seasonal campaigns.

### 3. A/B Testing Colors
Test different color schemes to see which converts better:
- Try warmer colors (orange, red) vs cooler (blue, green)
- Test high contrast vs subtle contrast
- Monitor conversion rates
